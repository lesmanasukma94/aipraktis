Website ini menggunakan prinsip:

- Hub & Spoke Architecture
- Topic Cluster SEO
- Internal Linking Graph
- Scalable Content System
- Zero Rework Structure

# 2. Struktur URL (FINAL)

/
→ Landing utama website

---

## 2.2 Blog Index

## 4.2 Vendor Hub

# 6. Internal Linking Rules

Setiap halaman WAJIB memiliki:

- Parent Link (kategori)
- 3–8 Related Articles
- 2 Vendor Links
- 1 CTA Monetization

---

# 7. Content Flow System

Homepage
→ Category Hub
→ Article
→ Vendor Hub
→ Landing Page
→ Conversion

---

# 8. SEO Architecture

Setiap halaman wajib:

- Static SEO Title
- Meta Description
- OG Image
- Schema Markup
- Breadcrumb Schema
- Internal Links

---

# 9. Scaling Strategy

Website harus siap:

- 5.000+ artikel
- 100+ kategori cluster
- 300+ vendor pages
- 1.000+ tools pages

Tanpa perubahan struktur URL.

---

# 10. Anti Rework Rule

Tidak boleh:

- Mengubah struktur URL setelah live
- Mengubah kategori setelah implementasi
- Menghapus hub utama
- Mengubah arsitektur navigasi

Semua harus FINAL dari awal.

---

# 11. Revenue Architecture Integration

Setiap halaman harus mendukung salah satu:

- Affiliate revenue
- Product revenue
- Lead generation
- Newsletter growth
- Consultation funnel

Tidak ada halaman yang “non-monetized”.

---

# 12. Performance Requirement

- Mobile-first
- Fast load
- Minimal JS
- Static generation (Astro preferred)
- SEO optimized rendering

---

# 13. Final Principle

Architecture ini dibuat untuk:

Build Once.

Scale Forever.

No Rework.

Revenue First.

SEO First.
